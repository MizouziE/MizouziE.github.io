What's the point?

The whole point of this is to force me to be comfortable coding in front of others. This is kind of my learning style, and it seems to be working.

I figure, if I do this, in this way, I'll be more comfortable much quicker and therefore more productive.

Please bear in mind that I am a PHP/Laravel developer first, so my actual abilities are probably beyond what I have on display here, being restricted to HTML/CSS/JavaScript.

My experience:

  • Here are some examples of sites I have worked on. More so on the backend API, but I've done small adjustments and am familiar with the frontend also. They are all instances of the same frontend framework and are served by by a single API.
    • Ladbrokes blog Content managed by publishers using the custom CMS attached to the API.
    • Party Casino blog Same as above, but also with different locale options.
    • Gioco Digitale blog Still served by the same API, but with slight customizations to the frontend structure.
  • What I'm currently working on is rebuilding a wordpress based learning portal on a Laravel framework. Like the above examples there will be an admin panel which will be made up of a number of Livewire components.
  • I have a few personal projects online at the moment too, you'll find links to them in the navigation bar at the top of the page:
    • Enkle: This is a working project, originally developed on Laravel but migrated to simple HTML/CSS/JS to be hosted as a stand-alone site. A child-friendly version of the popular word game Wordle. Try to find the word within the number of allowed guesses.
    • Minerals: A searchable web-app that uses a US Government API to show micro nutritional values for different foods. An additional API is called to provide images for each item. I am building this on a Laravel framework and use Tailwind CSS to help polish the front end.

What I want to include in all this:

  • The about me section needs to clearly show that I like PHP development more regarding actual content
  • However, that section needs to have cool feature (NOTE: not necessarily useful, just cool) to show examples of my ability to actually code
  • Custom-ish JavaScript functionality that makes the section more than "Read-only"
  • Also some kind of scroll induced animation would be very nice
  • Styling, of course, needs to be on point and I'll likely implement tailwindCSS for this cos I like working with it
  • Write a nice solution for light/dark theme switching
  • Some incorporation of an API call just to show I can do it
  • A display of my 3 most recent tweets
  • Direct email form and submition so users can leave any comments or suggestions. Input is ALWAYS welcome!
